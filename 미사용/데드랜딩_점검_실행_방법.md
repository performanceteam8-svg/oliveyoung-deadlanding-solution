# 데드랜딩 점검 스크립트 로컬 실행 방법

## 1. 터미널 열기

### Windows
- **방법 A**: `Win + R` 누른 뒤 `cmd` 입력 후 Enter → 명령 프롬프트(검은 창) 열림  
- **방법 B**: `Win + R` 누른 뒤 `powershell` 입력 후 Enter → PowerShell 창 열림  
- **방법 C**: 시작 메뉴에서 "명령 프롬프트" 또는 "Windows PowerShell" 검색 후 실행  
- **방법 D**: VS Code / Cursor 하단의 **터미널** 탭 클릭 (이미 프로젝트 폴더일 수 있음)

---

## 2. 프로젝트 폴더로 이동

터미널에 아래를 **한 줄씩** 입력하고 Enter를 누릅니다.

```text
cd C:\Users\MADUP\vibecoding
```

- `cd` = "이 폴더로 이동"
- 경로가 다르면 본인 PC의 실제 경로로 바꿉니다.  
  (예: `cd D:\projects\vibecoding`)

**확인**: 다음을 입력해 보세요.

```text
dir
```

- `데드랜딩 솔루션 시트.csv`, `dead_landing_check.py` 등이 보이면 올바른 폴더입니다.

---

## 3. Python 설치 여부 확인

```text
python --version
```

- `Python 3.x.x` 같이 나오면 정상입니다.
- `'python'은(는) 인식되지 않습니다` 라고 나오면 Python을 먼저 설치해야 합니다.  
  → [python.org](https://www.python.org/downloads/) 에서 설치 후, 설치 시 **"Add Python to PATH"** 체크.

---

## 4. 점검 스크립트 실행

### 251101(2025-11-01) 이후 행만 점검하고 결과 CSV 저장

```text
python dead_landing_check.py --since 251101 --suffix "(251101)"
```

- 한 줄 전체를 복사해서 터미널에 붙여넣은 뒤 **Enter**를 누릅니다.
- 실행 중에는 "점검 중" 메시지 없이 잠시 멈춘 것처럼 보일 수 있습니다. (URL을 하나씩 검사하는 동안)
- 끝나면 터미널에 **데드랜딩 점검 결과** 요약이 출력됩니다.

---

## 5. 실행 후 확인

- **결과 파일 위치**: 같은 폴더  
  `C:\Users\MADUP\vibecoding\데드랜딩_솔루션_시트_점검결과(251101).csv`
- **열기**:  
  - 탐색기에서 해당 폴더 열고 `데드랜딩_솔루션_시트_점검결과(251101).csv` 더블클릭  
  - 또는 엑셀에서 **파일 → 열기** 후 위 CSV 선택
- **결과 보는 열**:  
  - **V열** = E열(영국) 링크 검증 결과 (정상 / 오류)  
  - **W열** = F열(이스라엘) 링크 검증 결과 (정상 / 오류)

---

## 6. 중복 제거 방식으로 실행 (새 시트에 영국/이스라엘 각각 수집 후 점검)

Link 탭의 **모든** E열(영국)·F열(이스라엘) URL을 수집해 **중복 제거**한 뒤 점검하고, 결과를 **새 CSV(새 시트)** 로 저장하려면 아래 스크립트를 사용합니다.

```text
python dead_landing_check_deduped.py
```

- **결과 파일**: `데드랜딩_링크_점검결과_중복제거.csv`
- **구조**:  
  - 위쪽: **영국** – URL 목록(중복 제거) + 검증결과(정상/오류)  
  - 빈 행  
  - 아래쪽: **이스라엘** – URL 목록(중복 제거) + 검증결과(정상/오류)
- **파일명에 접미사 붙이기** (예: `(2차)`):  
  `python dead_landing_check_deduped.py --suffix "(2차)"`  
  → `데드랜딩_링크_점검결과_중복제거(2차).csv` 로 저장됩니다.

---

## 7. 다른 조건으로 실행할 때

| 실행 목적 | 입력할 명령 |
|----------|-------------|
| 251101 이후 + 결과 파일명 (251101) | `python dead_landing_check.py --since 251101 --suffix "(251101)"` |
| 250101 이후 + 결과 파일명 (2차) | `python dead_landing_check.py --since 250101 --suffix "(2차)"` |
| 최근 90일 이내만 점검 (기본 파일명) | `python dead_landing_check.py` |
| 테스트용 5행만 점검 | `python dead_landing_check.py --limit 5` |

- `--since YYMMDD` : 이 날짜 **이후** 시작일인 행만 점검  
- `--suffix "문자열"` : 결과 CSV 파일명 맨 뒤에 붙일 문자열 (예: `(251101)`, `(2차)`)  
- `--limit 숫자` : 점검할 최대 행 수 (테스트용)

---

## 8. 자주 나오는 상황

- **"python을 찾을 수 없습니다"**  
  → Python 설치 후, **설치 시 "Add Python to PATH"** 선택했는지 확인.  
  → 그래도 안 되면 터미널을 완전히 닫았다가 다시 열고, `python --version` 다시 시도.

- **실행은 되는데 결과 CSV가 안 보임**  
  → 터미널에 나온 "결과 저장: ..." 다음 줄에 적힌 파일 경로를 확인.  
  → `cd` 로 이동한 폴더와 실제 스크립트/CSV가 있는 폴더가 같은지 확인 (`dir` 로 목록 확인).

- **실행이 오래 걸림**  
  → 251101 이후 행이 많으면 URL 하나씩 검사해서 수 분~십 수 분 걸릴 수 있습니다.  
  → 터미널을 닫지 말고 완료될 때까지 기다리면 됩니다.

이 순서대로 하시면 로컬 터미널에서 데드랜딩 점검을 실행하고, 결과 CSV를 확인할 수 있습니다.
